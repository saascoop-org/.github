# Contributing to SaaSCoop

Thank you for your interest in contributing to SaaSCoop.

SaaSCoop is an open source cooperative initiative focused on digital solutions, artificial intelligence, education and technology as a common good.

## Ways to contribute

You can contribute by:

- reporting bugs
- suggesting ideas
- improving documentation
- translating content
- testing prototypes
- creating tutorials
- reviewing pull requests
- improving accessibility
- participating in discussions

## Before contributing

Please read:

- `README.md`
- `CODE_OF_CONDUCT.md`
- `GOVERNANCE.md`
- the repository-specific documentation

## Opening an issue

Use issues to:

- report bugs
- suggest improvements
- propose features
- ask for clarification
- document problems

A good issue should include:

- clear title
- context
- expected behavior
- current behavior
- screenshots or logs, when useful
- possible solution, if known

## Pull requests

Before opening a pull request:

1. Check if there is an existing issue.
2. Explain the purpose of the change.
3. Keep changes focused.
4. Update documentation when needed.
5. Avoid unrelated changes in the same pull request.

## Documentation contributions

Documentation is a core part of SaaSCoop.

You can help by improving:

- README files
- tutorials
- architecture notes
- onboarding guides
- glossary
- translations
- examples

## Language

SaaSCoop may publish content in:

- English
- Portuguese
- Spanish

English is preferred for repository structure and technical interoperability.

Portuguese and Spanish are welcome, especially for educational and community materials.

## Community behavior

All contributors must follow the Code of Conduct.

We value respectful, constructive and inclusive collaboration.

## Legal and licensing

By contributing, you agree that your contribution may be distributed under the license used by the repository.

Do not submit copyrighted material that you do not have permission to contribute.

## Maintainer review

Maintainers may request changes before merging.

Pull requests may be declined if they are outside the project scope, legally risky, poorly documented or inconsistent with SaaSCoop principles.
