# Security Policy

## Supported projects

SaaSCoop is currently in an early structuring phase.

Security support may vary by repository and project maturity.

Each repository should define its own security expectations when necessary.

## Reporting a vulnerability

If you discover a security vulnerability, please do not disclose it publicly before maintainers have had time to review it.

Please contact the maintainers through the official contact channel listed in the organization profile or repository documentation.

When reporting, include:

- affected repository
- description of the issue
- steps to reproduce
- potential impact
- suggested fix, if available

## Responsible disclosure

We ask contributors and users to act responsibly and give maintainers reasonable time to investigate and fix issues.

## Security scope

Security concerns may include:

- exposed credentials
- unsafe dependencies
- authentication flaws
- data leakage
- insecure configuration
- prompt injection risks in AI systems
- privacy risks
- unsafe handling of user-generated content

## No sensitive data

Do not submit real personal data, private credentials, confidential documents or production secrets to public repositories.

## AI safety and privacy

Projects involving AI should avoid storing or exposing sensitive user data.

Whenever possible, AI systems should document:

- data sources
- limitations
- risks
- intended use
- privacy considerations
